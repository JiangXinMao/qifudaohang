祈福导航系统 V1.5 安装与升级说明

一、运行环境

1. PHP 8.2 或更高版本，并启用 PDO、PDO_MySQL、cURL、OpenSSL、fileinfo 扩展。
2. MySQL 5.6+、MySQL 5.7/8.0 或 MariaDB 10.x。
3. Apache 或 Nginx，生产环境必须启用 HTTPS。

二、全新安装

1. 将程序上传到独立站点目录，不要把数据库备份或 SQLite 文件放在网站根目录内。
2. 浏览器访问 /install/，按向导填写数据库连接和管理员信息。
3. 管理员账号和密码均必须为 6-18 位英文字母或数字。全新安装默认账号和密码为 admin1 / 123456。
4. 安装完成后确认 install/install.lock 已生成，并删除不再需要的安装写权限。
5. 立即将网站根目录中的 admin 文件夹重命名为不易猜测的名称；程序会自动识别新路径，不要保留默认 admin 目录的副本。
6. 按《安全部署说明.md》限制数据库端口、账号权限和后台访问来源。

三、V1.5 主要更新

1. 数据库访问统一使用参数化查询，阻断 SQL 注入和堆叠语句。
2. 默认禁止网页数据库恢复和 Web 运行期 DDL，移除后台恢复入口。
3. 管理员密码使用 password_hash 安全散列，登录使用 Session 身份验证。
4. 登录、密码修改和后台写操作增加 CSRF 校验。
5. 登录失败达到 5 次后，对账号与来源 IP 锁定 15 分钟，降低爆破风险。
6. 管理员账号和密码均限制为 6-18 位英文字母或数字。
7. 加固 Cookie、Host Header、代理 IP、定时任务密钥和 SSRF 防护。
8. 上传图片同时校验扩展名、MIME、真实图像内容和随机文件名。
9. 禁止外部访问配置、备份、测试、SQLite、旧扫描器和安装锁文件。
10. jQuery 升级到 3.6.0，修复已知旧版本前端安全风险。
11. 前台页脚在内容较少时保持页面底部，兼容桌面端和 H5。
12. 新增自动化安全回归测试，版本号统一升级为 V1.5（内部版本 1500）。
13. 新增公安网安备案设置，支持备案号、HTTPS 查询链接及前台盾牌组件，兼容桌面端与 H5 页脚换行。
14. 新增网站在线统计组件与后台开关，支持真实匿名访客数据和按日稳定的随机展示数据。
15. 新增前台本站搜索开关，开启后默认筛选本站导航资源，关闭后恢复百度为默认搜索引擎。

四、升级说明

1. 升级前通过服务器终端完成数据库与旧程序目录备份。
2. 不要覆盖现有 config.php；先对照 V1.5 配置项，再迁移数据库连接信息。
3. 旧版明文或 MD5 管理员密码在首次成功登录后会自动升级为安全散列。
4. 升级后立即修改管理员密码，检查后台登录、站点管理、友链审核和图片上传。
5. 不要重新开放“新后台目录/restore.php”；数据库恢复只能由运维人员在服务器终端执行。
