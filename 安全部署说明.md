# 祈福导航系统安全部署

应用已经默认关闭网页数据库恢复，并禁止 Web 运行期执行 DROP、ALTER、CREATE、TRUNCATE、ATTACH 等破坏性数据库语句。生产环境仍必须完成以下基础设施配置；应用代码无法阻止直接暴露在公网的 MySQL 端口被扫描。

## MySQL

1. MySQL/MariaDB 只监听 `127.0.0.1` 或内网管理地址，禁止监听公网地址。
2. 云安全组和系统防火墙禁止公网访问 `3306`。
3. 为本系统创建独立数据库账号，Host 限制为 `localhost`，不要使用 `root`。
4. 安装完成后，运行账号只授予业务表的 `SELECT, INSERT, UPDATE, DELETE` 权限，不授予 `DROP, ALTER, CREATE, FILE, PROCESS, GRANT`。
5. 数据库备份通过后台下载后离线保存；恢复操作只允许在服务器终端由运维人员执行。

## Web 服务器

Apache 会读取项目自带的 `.htaccess`。Nginx 需要在站点配置加入：

```nginx
location ~ ^/(backup|tests|\.local|includes/(sqlite|360safe))(/|$) { deny all; }
location = /config.php { deny all; }
location = /install/install.lock { deny all; }
```

站点必须启用 HTTPS，并将 PHP 的 `display_errors` 设为 `Off`、`log_errors` 设为 `On`。部署完成后确认 `install/install.lock` 存在。

将 `config.php` 中的 `QIFU_CANONICAL_HOST` 改为正式域名，例如 `nav.example.com`，防止 Host Header 污染系统生成的链接。

## 管理后台

安装完成后必须将网站根目录中的 `admin` 文件夹重命名为不易猜测的名称，不要保留默认目录副本。程序会自动识别新后台路径，内部菜单、表单、AJAX 和 CSRF 校验无需额外配置。管理员账号和密码均只能使用 6-18 位英文字母或数字。连续 5 次登录失败会锁定对应 IP 与账号 15 分钟。建议再通过防火墙、VPN 或 Web 服务器白名单限制“新后台路径”的来源地址，并确认旧 `/admin/` 返回 404。
